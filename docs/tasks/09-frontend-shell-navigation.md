# Task 09: Frontend Shell and Application Navigation

## Whole-picture context

The frontend should make the product feel like a real operator tool, not a toy.

## Specific task goal

Create the basic Next.js application shell, layout, navigation, and API integration foundation.

## Requirements

- Create routes or pages for Dashboard, Uploads, Issues, Copilot, and Cases.
- Add a shared layout and navigation.
- Add an API client layer.
- Add basic loading and error states.
- Use a clean, operator-friendly UI rather than a marketing-style UI.

## Output

A navigable frontend shell wired to the backend.

## Acceptance criteria

- App routes render cleanly.
- Navigation works.
- The API base URL is configurable.
- Placeholder states are sensible.
