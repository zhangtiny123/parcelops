# Task 11: Issues List, Dashboard, and Issue Detail UI

## Whole-picture context

The dashboard proves business value. Issue detail proves explainability.

## Specific task goal

Implement the operator-facing dashboard, issue list, filters, and issue detail pages.

## Requirements

- The dashboard shows total recoverable amount, issue counts by type, issue counts by provider, and recent trend.
- The issues page supports filters.
- The issue detail view shows the issue summary, linked shipment, invoice, or order context, readable evidence payload, recoverable amount, and confidence.
- Keep the UI clean and fast.

## Output

A believable recovery operations dashboard.

## Acceptance criteria

- The dashboard renders real data.
- Filters narrow the issue list correctly.
- Issue detail makes the anomaly understandable to a human.
