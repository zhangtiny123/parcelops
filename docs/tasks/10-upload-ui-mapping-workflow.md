# Task 10: Upload UI and Mapping Workflow

## Whole-picture context

The product begins with messy source files, so upload and mapping are critical to the demo.

## Specific task goal

Implement the frontend flow for file upload, preview, and schema mapping.

## Requirements

- The upload page supports file selection and submission.
- Show the uploaded file list and statuses.
- Show preview rows.
- Show suggested mappings.
- Allow user edits to mappings.
- Save mappings to the backend.
- Allow triggering normalization from the UI.

## Output

A complete upload-to-normalization UI flow.

## Acceptance criteria

- A user can upload a file from the browser.
- A user can review preview rows.
- A user can save a mapping.
- A user can start normalization.
- Status updates are visible in the UI.
