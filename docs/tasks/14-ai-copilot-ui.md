# Task 14: Copilot UI

## Whole-picture context

The copilot is a visible signal that the system is intelligent, but it must stay tightly connected to evidence.

## Specific task goal

Build the frontend chat interface for the AI copilot.

## Requirements

- Add a chat page with message history.
- Stream responses if practical, otherwise use standard request and response behavior.
- Show cited issue IDs or record references where appropriate.
- Allow canned starter prompts for top recoverable issues, spend increase analysis, and highest-confidence billing errors.
- Keep the UI useful and restrained.

## Output

A productized chat interface for operational analysis.

## Acceptance criteria

- A user can ask grounded questions.
- Responses are shown clearly.
- References are visible enough to inspect the answer.
